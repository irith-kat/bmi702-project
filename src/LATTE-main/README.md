## Run Model

Run `a_train_final.py` directly, or via the `run_latte()` wrapper in `src/latte/latte.py`.

### Dependencies

```
tensorflow>=2.x  (tested with Keras 3)
scikit-learn
pandas
numpy>=2.0
```

### Key arguments

| Argument | Description |
|----------|-------------|
| `--train_directory` / `--train_filename` | Directory and filename of the labeled training CSV |
| `--test_directory` / `--test_filename` | Directory and filename of the test CSV |
| `--unlabel_directory` / `--unlabel_filename` | Directory and filename of the unlabeled CSV |
| `--embedding_filename` | Code embedding CSV (rows = components, columns = code names) |
| `--key_code` | Comma-separated key codes, e.g. `"PheCode:428.1,PheCode:428.2"` — must have embeddings in the embedding file |
| `--colums_min` / `--colums_max` | Column index range of the covariate features (default 3 / n_features+3) |
| `--embedding_dim` | Dimension of code embeddings |
| `--flag_cross_dataset` | `1` = use separate train/test files as-is; `0` = cross-validate within `train_filename` using `--number_labels` patients for training |
| `--number_labels` | Number of labeled patients to use for training when `flag_cross_dataset=0` |
| `--epochs` | Total training epochs |
| `--epoch_silver` | Epochs of pre-training on silver labels (PheNorm-style, derived from `key_code` counts) |
| `--layers_incident` | GRU layers and units for temporal modelling: `"80"` = one layer of 80 units; `"80,90"` = two layers |
| `--weight_prevalence` | Loss weight for the EVER/NEVER prevalence head |
| `--weight_unlabel` | Loss weight for unlabeled data (silver-label joint training) |
| `--weight_constrastive` | Loss weight for contrastive representation learning |
| `--weight_smooth` | Loss weight for temporal smoothness regularization |
| `--weight_additional` | Loss weight for additional semi-supervised regularization |
| `--flag_save_attention` | `1` = save learned visit/code attention weights for visualization |
| `--flag_load_model` | `1` = reload previously saved model from `save_directory` |
| `--flag_train_augment` | `1` = augment training with ±1-month time windows in addition to the default window |

**Notes:**
- For MS prediction, use smaller `weight_smooth` and `weight_constrastive` (~0.05); two GRU layers may help.
- For HF/T2D prediction, a single GRU layer (`layers_incident="80"`) is sufficient.



## Compatibility Changes

The following changes were made to run LATTE on Python 3.12 / NumPy 2.x / Keras 3. The model architecture and training logic are unchanged.

### `a_utilize_semi.py` — NumPy 2.x
- Replaced removed aliases `np.int` → `np.int64` and `np.float` → `np.float64`.

### `a_Transformer.py` — Keras 3
- Replaced `tf.matmul`, `tf.math.*`, `tf.nn.*` ops with `keras.ops` equivalents, which are required when building a functional `Model` with Keras 3's `KerasTensor` type.
- Fixed `tf.transpose(..., transpose_b=True)` pattern: replaced with explicit `kops.transpose(k, axes=[..., ndim-1, ndim-2])` using non-negative absolute indices (TF's `InvertPermutation` gradient op rejects negative indices).

### `a_semi_model_final.py` — Keras 3 + NumPy 2.x
- Replaced `tf.concat`, `tf.squeeze`, `tf.nn.*`, `tf.matmul`, `tf.expand_dims` with `keras.ops` equivalents inside the functional model build (`code_attention`, `Model_prediction`).
- Replaced `/` with `_` in Keras layer names (Keras 3 rejects `/` in names).
- Squeezed prevalence predictions and labels to consistent 1-D `(batch,)` shape: Keras 3's `binary_crossentropy` enforces equal rank, whereas Keras 2 broadcast silently.
- Removed `tf.expand_dims` from `prediction_MLP_total` and fixed the MLP contrastive `Y_half_batch` computation accordingly.
- Fixed `int(weights_test_total[samplei])` → `.flat[0]` to handle NumPy 2.x's stricter scalar conversion rules.
- Changed `model.save(path)` to use a `.keras` extension, required by Keras 3.

### `a_train_final.py` — pre-existing bug
- Fixed argument order in all three `get_data_from_csv` calls (positional args `dirr_data` and `filename` were swapped, which would have caused a `FileNotFoundError` in any environment).

### Python wrapper (`src/latte/`)
A lightweight Python wrapper (`src/latte/latte.py`, `src/latte/embeddings.py`) was added to integrate LATTE into the broader BMI702 phenotyping pipeline. It provides:
- `format_latte_input()` — converts a long-format observation log + gold labels into LATTE's wide CSV format.
- `build_cooccurrence_embeddings()` — builds code embeddings via truncated SVD on a patient × code co-occurrence matrix.
- `run_latte()` — calls `a_train_final.py` via subprocess and returns predictions as a DataFrame.
- `compute_abcgain()` — evaluates predictions with AUC, F1 @ 95% specificity, and ABCgain.
